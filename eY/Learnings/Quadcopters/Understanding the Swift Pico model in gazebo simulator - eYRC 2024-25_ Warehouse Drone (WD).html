Understanding the Swift Pico model in Ignition Gazebo

In this tutorial you will learn how to use the Swift Pico drone model for Task 1B. You need to complete the Installations part of Task 1B to continue further.

  • Navigate to your workspace.

    cd pico_ws

  • Set the GZ_SIM_RESOURCE_PATH environment variable to include the path to the rotors_swift_description/models directory.

    export GZ_SIM_RESOURCE_PATH="$HOME/pico_ws/src/rotors_simulator/rotors_swift_description/models"

  • Launch the following command to see the gazebo world. Make sure there are no multiple gazebo windows opens.

    ros2 launch rotors_swift_gazebo swift_pico_simulation.launch.py

    NOTE: To kill it use CTRL+C (Kill command) rather than CTRL+Z (hibernating command), it will take some time but wait patiently.

    NOTE: Create a new terminal in the workspace and do not forget to source the terminal everytime.

    source install/setup.bash

  • Check all the topics published by gazebo. Run the following command in the terminal to check all the topics published by Gazebo

    ros2 topic list

    You should find the topic “/drone_command”

  • Check the type of messages accepted by the “/drone_command” topic. “/drone_command” is a topic subscribed by the swift pico model. It commands the drone’s motion in terms of roll, pitch, yaw and throttle. Run the following command in the terminal to check:

    ros2 topic info /drone_command

    Your output will display the topic type as “swift_msgs/msg/SwiftMsgs”. Check the structure of the message by typing the following command in the terminal:

    ros2 interface show swift_msgs/msg/SwiftMsgs

    The values for rcRoll, rcPitch, rcYaw and rcThrottle range from 1000 to 2000

  • Arming the Drone:

    An armed drone means the drone is ready to take commands from a user or software to fly. The condition to arm the drone is rcThrottle = 1500 and rcAUX4 = 1500. To test arming the drone model, publish the following message to the topic “/drone_command” by typing the command :

     ros2 topic pub /drone_command swift_msgs/msg/SwiftMsgs "rc_roll: 1500
rc_pitch: 1500
rc_yaw: 1500
rc_throttle: 1500
rc_aux1: 0
rc_aux2: 0
rc_aux3: 0
rc_aux4: 1500
drone_index: 0"

    The propellers should start rotating.

    NOTE: After publishing any command to the drone, before entering new command press Ctrl+C and then enter next command.

  • Flight (Take-Off):

    The condition for the drone to take-off is rcThrottle ≥ 1550, after arming. To test the drone’s take- off, publish the following message to increase the throttle:

    ros2 topic pub /drone_command swift_msgs/msg/SwiftMsgs "rc_roll: 1500
rc_pitch: 1500
rc_yaw: 1500
rc_throttle: 1550
rc_aux1: 0
rc_aux2: 0
rc_aux3: 0
rc_aux4: 1500
drone_index: 0"

    The drone should now steadily rise until a new command is given, as shown in figure 4.

    NOTE: If the drone goes up and gets stuck to the camera, you can bring it down by disarming the drone

  • Disarming the Drone:

    A disarmed drone means the drone is in stop state. The condition to disarm the drone is rcThrottle = 1000. To test disarming the drone model, publish the following message to the topic “/drone_command” by typing the command:

    ros2 topic pub /drone_command swift_msgs/msg/SwiftMsgs "rc_roll: 1500
rc_pitch: 1500
rc_yaw: 1500
rc_throttle: 1000
rc_aux1: 0
rc_aux2: 0
rc_aux3: 0
rc_aux4: 1500
drone_index: 0"

    The drone should now be disarmed. And come back to the ground.
