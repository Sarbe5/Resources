Welcome to Turtlesim 2 learning resources

Turtlesim is the Flagship example application for ROS and ROS 2. It demonstrates in simple but effective ways the basic concepts.

Start the Turtle Simulator

  • To start turtlesim, enter the following command in your terminal:

    ros2 run turtlesim turtlesim_node

    In the terminal, under the command, you will see messages from the node.

    [INFO][turtlesim]: Starting turtlesim with node name /turtlesim
[INFO][turtlesim]: Spawning turtle [turtle1] at x=[5.000000], y=[2.000000], theta=[0.000000]

  • Open a new terminal and source ROS 2 again. Now you will run a new node to control the turtle in the first node.

    ros2 run turtlesim turtle_teleop_key

Description of Topics

  • To list the ROS 2 topics, use this command,

    ros2 topic list

    You will get the following topics:

    • /turtle1/cmd_vel : Used to control the turtle's velocity.

    • /turtle1/color_sensor : Used to get the color sensor data from the turtle.

    • /turtle1/pose : Used to get the pose (position and orientation) of the turtle.

  • Running these commands in separate terminals will let you monitor the data being published to each topic in real time.

    ros2 topic echo /<topic_name>

Description of Services

  • To list the ROS 2 services, use this command,

    ros2 service list

    You will get the following services:

    • /clear : clears the background of trail lines
    • /kill : to kill a turtle
    • /reset : resets the position of the turtle
    • /spawn : to spawn a turtle
    • /turtle1/set_pen : sets the color and thickness of the trail line.
    • /turtle1/teleport_absolute : move the turtle instantly

    You can ignore the services with the word parameter in the names for now.

  • The /spawn service in turtlesim allows you to create a new turtle at a specified location with a given orientation. To call this service, you can use the ROS 2 service call command

    ros2 service call -r 0.5 /spawn turtlesim/srv/Spawn "{x: 5,y: 5,theta: 0}"

  • You can explore other services also.

Description of Actions

  • To list the ROS 2 actions, use this command,

    ros2 action list

    You will get the following actions:

    • /turtle1/rotate_absolute

  • You can send a goal to an action server and see the feedback in real-time:

    ros2 action send_goal /turtle1/rotate_absolute turtlesim/action/RotateAbsolute {'theta: 1.57'} --feedback

  • When you send a goal using the above command, you will see feedback messages as they are published. These feedback messages provide information on the progress of the action.

Interacting

In this step you will practice interacting with the topics, services, actions and parameters provided by the turtlesim.

Open a new terminal and source ROS 2 again

  • To Draw a Line

    Use this command to draw a Line

    ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "linear:
  x: 2.0
  y: 0.0
  z: 0.0
angular:
  x: 0.0
  y: 0.0
  z: 0.0"

  • To Draw a Circle

    Use this command to draw a Circle

    ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "linear:
  x: 2.0
  y: 0.0
  z: 0.0
angular:
  x: 0.0
  y: 0.0
  z: 1.8"

  • Remapping:

    When you a spawn an another turtle. You need a second teleop node in order to control turtle2. However, if you try to run the same command as before, you will notice that this one also controls turtle1 . The way to change this behavior is by remapping the cmd_vel topic.
    ros2 run turtlesim turtle_teleop_key --ros-args --remap turtle1/cmd_vel:=turtle2/cmd_vel

    Open this in new terminal

    ros2 topic pub /turtle2/cmd_vel geometry_msgs/msg/Twist "linear:
  x: -2.0
  y: -2.0
  z: 0.0
angular:
  x: 0.0
  y: 0.0
  z: 1.0"
